# 数学文献 PDF 翻译工具 — 阶段汇报

> 汇报时间：2026-03-27

---

## 一、做了什么

开发了一个本地运行的网页工具，用于将德语、法语、英语的数学文献 PDF 翻译为中文，核心目标是**在翻译的同时完整保留所有 LaTeX 数学公式**，输出结果可以直接在浏览器中阅读，公式实时渲染显示。

---

## 二、核心思路

### 难点所在

数学文献的翻译与普通文本翻译有本质区别：文中大量的公式（如 $\varepsilon$-$\delta$ 语言、积分、矩阵等）必须原样保留，不能被翻译或改动。普通翻译工具会破坏公式结构，导致输出乱码或公式丢失。

### 解决方案

采用**双模型并行翻译 + 交叉审核合并**的流程：

```
输入 PDF
   │
   ├─ 文字型（能直接提取文本）─→ 直接读取文字，按段落分批
   └─ 扫描件（图片型）────────→ 先将每页渲染成图片，再用视觉识别
                                               │
                          ┌────────────────────┤
                          ▼                    ▼
                    Claude 翻译           GPT-4o 翻译
                    （独立完成）          （独立完成）
                          └────────────────────┘
                                      │
                                      ▼
                             Claude 作为评审员
                          对比两份译文，按优先级合并：
                          ① 公式是否完整
                          ② 数学逻辑是否准确
                          ③ 术语是否规范
                          ④ 语言是否流畅
                                      │
                                      ▼
                          输出中文网页（公式实时渲染）
```

让两个模型独立翻译同一段内容，再由第三方（Claude）做评审，可以互相纠错，翻译质量明显优于单模型。

## 四、遇到的困难

### 1. 公式保护问题

翻译模型有时会"顺手"翻译公式内部的内容，如将 $\varepsilon$ 写成"ε"，或改动上下标结构。解决方法是在系统提示词中用强规则约束（绝对不翻译 `$...$` 内的内容），并通过交叉审核环节专门检查公式完整性。

### 2. 大文件扫描件处理

《数学分析讲义（第三册）》是 32 MB 的扫描版 PDF，共约 200 页，需要将每一页渲染成图片后逐批发送给模型识别和翻译。目前该任务尚未完成，原因是：

- **GPT-4o 账户欠费**（429 错误），导致并行翻译退化为单模型
- **API 访问权限问题**（403 错误），与本地网络代理配置有关，已排查修复

代理问题已解决，待 GPT-4o 配额补充后可重新运行。

### 3. 网络代理干扰

本地开发环境中，系统全局代理会拦截所有网络请求，包括对 `localhost` 的本地访问，导致网页样式无法加载、API 无法正常连通。已通过修改代理绕过规则解决。
